# FarMarT Sales System

FarMarT is a Java sales subsystem for a regional farm equipment and supplies business. It tracks people, stores, items, and invoices, then produces formatted sales reports for business analysis.

This project combines object-oriented design with relational database integration and custom data structures for sorting invoice data.

## Project Scope

The system supports:

- Parsing flat CSV data files into domain objects
- JSON conversion for core entities (people, stores, items)
- Database-backed data loading through JDBC (MySQL Connector/J)
- Sales reporting by customer, store, and total value
- Invoice sorting using a custom generic linked list implementation

Out of scope:

- The web-based customer storefront and online checkout platform

## Core Business Rules

- Product tax: 7.15% of product subtotal
- Service tax: 3.45% of service subtotal
- Equipment purchase: flat purchase pricing
- Equipment lease tax tiers:
  - Under $10,000: $0 tax
  - $10,000 to $100,000: $500 tax
  - Over $100,000: $1,500 tax

## Tech Stack

- Java (object-oriented design)
- MySQL (RDBMS)
- JDBC (database integration)
- Gson (JSON serialization)

Libraries included in `lib/`:

- `gson-2.10.1.jar`
- `mysql-connector-j-8.0.32.jar`

## Repository Layout

- `src/com/fmt/`: application-level classes (entry points and utilities)
- `src/framework/fmt/`: domain model, parsing, loading, and reporting
- `src/list/fmt/`: custom generic linked list and comparators
- `data/`: input CSV files, JSON outputs, and sample output
- `test/`: test package structure

## Build and Run

### Prerequisites

- Java 8+
- Terminal from repository root

### Compile

```bash
mkdir -p bin
javac -cp "lib/*" -d bin $(find src -name "*.java")
```

### Run Reports

```bash
java -cp "bin:lib/*" com.fmt.InvoiceReport
```

This runs the reporting flow and prints sorted sales summaries to standard output.

## Data and Environment Notes

- CSV input files are in `data/` (`Persons.csv`, `Stores.csv`, `Items.csv`, `Invoices.csv`, `InvoiceItems.csv`).
- JSON outputs (`Persons.json`, `Stores.json`, `Items.json`) are generated by conversion utilities.
- Database-backed loading is configured in `DatabaseConnection` and expects a reachable MySQL instance.

## Testing and Validation

- Sample datasets are provided under `data/` for component-level validation.
- `data/output.txt` can be used as a reference for report formatting expectations.
